Core development has focused on three areas, and we have included many other great improvements contributed by our open source community.</p> <p>You’ll notice the first major change before you even start the playtest: the “OpenRA” launcher is no more, and has been replaced with individual launchers for Red Alert, Tiberian Dawn, and Dune 2000. This is part of a major reorganisation of the project structure to improve backwards compatibility and modding support.</p> <div style="text-align:center"> <p><img src="/images/news/20170722-launchers.png" alt="New launchers" /></p> <p>Individual mods are now launched directly from your Desktop / Application Launcher.</p> </div> <p>The in-game mod chooser has been removed, but it is still possible to switch mods when joining a server from the Multiplayer list. Even better, if you have multiple versions installed (e.g. Playtest and Release, which now install to different locations on Windows) the server list will show games from <em>all</em> versions, and will switch to the correct version when you try to connect.</p> <p>The second focus area relates to how you control your units on the battlefield. For many years we have supported features like Attack Move, unit stances and queuing orders using hotkeys, but these have not always been obvious or intuitive to use. We have added a long-planned command bar interface for these, and made some adjustments and bug fixes to make them better and behave more like the original games.</p> <div style="text-align:center"> <p><img src="/images/news/20170722-ra-commandbar.png" alt="Unit command bar" /></p> <p>A new bottom command bar is now available for issuing common orders and changing unit stances.</p> </div> <p>The final focus area fundamentally changes the way that buildings occupy space and receive damage, taking inspiration from the original Dune 2000, which was more advanced than the earlier (and some later!) games.</p> <div style="text-align:center"> <p><img src="/images/news/20170722-td-buildinghitshapes.png" alt="Building hit boxes" /></p> <p>New building physics improves the way that structures are targeted and take damage.</p> </div> <p>Some of the smaller but still important new features include:</p> <ul> <li>Automatic discovery of LAN games in the Multiplayer browser.</li> <li>Improvements to the Global Chat UI.</li> <li>New search and filtering options in the Map editor.</li> <li>Another 9 campaign missions for D2K, which is now half way complete!</li> <li>Missing maps can now be installed directly from the Replay browser.</li> <li>Significant performance and memory improvements when playing music.</li> </ul> <p>Please see the <a href="https://github.com/OpenRA/OpenRA/wiki/Changelog/f4a27f19459d2dd35d4bc08a75a741363f146422">full changelog</a> for information on many more improvements that we have not mentioned here.</p> <div style="text-align:center"> <p><img src="/images/news/20170722-d2k-editor.png" alt="New D2K missions" /></p> <p>It is now possible to search and filter actors and tiles in the map editor.</p> </div> <p>This playtest includes big changes which may have broken your favourite feature, and which have definitely affected the game balance in all three mods. This makes it more important than ever to play games on this and future playtests and let us know of any issues via <a href="http://www.sleipnirstuff.com/forum/viewforum.php?f=80">our forum</a>, our <a href="https://github.com/OpenRA/OpenRA/issues">GitHub issue tracker</a>, or in the comments below.</p> <p>Don’t forget, if you’re on Windows or macOS you can have playtest-20170722 and release-20170527 installed at the same time, and switch between the two when joining servers with other players. We have listened to the feedback from both players and modders, and have spent the last few months working on a comprehensive solution.</p> <p>We have done this by decoupling the game engine (OpenRA.Game.exe and support files) from our default mods, and have put <em>the mod</em> in charge. Mods now manage their own engine files, which means that they can exist <em>alongside</em> our official mods with no dependencies between them.</p> <p>Our new <a href="https://github.com/OpenRA/OpenRAModSDK">mod SDK</a> provides a development environment that simplifies the engine and dependency management, and allows you to generate proper installers for players to install your game.
