With some help from our <a href="https://forum.openra.net">Forum</a> and <a href="https://discord.openra.net">Discord</a> community, we are now ready to release <a href="/download/">playtest-20190825</a> for wider testing and feedback.</p> <p>The <a href="/news/preview-20190606/">Summer Release Preview</a> article outlined several of the new UI features, including <span style="color: #d92c2c">savegame support</span>, a new <span style="color: #d92c2c">spectator UI</span>, and a long planned <span style="color: #d92c2c">Tiberian Dawn interface overhaul</span>. Part of the overhaul has moved the familiar targeting lines that display when you command units into the “Waypoint Mode” (Shift) overlay by default. The previous behaviour can be restored through the Settings menu (Target Lines: Automatic) if you prefer.</p> <div style="text-align:center"> <p><img src="/images/news/20190825-waypoint.png" alt="Waypoint mode." width="600px" /></p> <p>The improved waypoint mode allows you to plan your attack strategies.</p> </div> <p>Other noteable changes to watch out for include:</p> <ul> <li>Force Move (Alt) can be used to undeploy Construction Yards and land helicopters</li> <li>Aircraft respond to Scatter orders from the Command Bar (Ctrl + X)</li> <li>Significantly improved behaviour for harvesters, aircraft, attack-move, and unit repair</li> <li>Aircraft-based support powers can be given an attack direction</li> <li>The controversial Allied Hind helicopter in RA has been replaced with the Black Hawk</li> <li>Two new-to-OpenRA missions from the RA Counterstrike expansion</li> <li>The D2k Death Hand missile now works like the original game</li> <li>A collection of <a href="https://github.com/OpenRA/OpenRA/wiki/Changelog/4e8c55f0a17caee09f0833a3fbf9fc0861b982f6#ra-balance-changes">Red Alert</a> and <a href="https://github.com/OpenRA/OpenRA/wiki/Changelog/4e8c55f0a17caee09f0833a3fbf9fc0861b982f6#td-balance-changes">Tiberian Dawn</a> balance changes</li> </ul> <p>As always, the <a href="https://github.com/OpenRA/OpenRA/wiki/Changelog/4e8c55f0a17caee09f0833a3fbf9fc0861b982f6">full changelog</a> is available with more information on the changes and fixes.</p> <div style="text-align:center"> <p><img src="/images/news/20190606-directional-powers.gif" alt="Directional powers." width="600px" /></p> <p>Aircraft-based support powers can be given a direction by dragging the mouse while targeting.</p> </div> <p>As a .NET application, OpenRA relies on the Mono runtime for our Linux and macOS players. Unfortunately, the Mono packages provided by some major LTS distros are several years out of date and incompatible with the modern .NET standard. Worse, the new Debian Buster and Ubuntu 19.04 releases shipped with a serious Mono bug that causes unavoidable crashes OpenRA and other .NET applications.</p> <p>Our solution for these issues is to package the parts of Mono that OpenRA requires inside the AppImage and macOS app releases. This means that you can finally play OpenRA without having to install any system-level dependencies. For players or packagers who prefer to compile OpenRA from source, Mono 5.4 or later and msbuild are now required.</p> <p>Updating our base requirements has allowed several further improvements:</p> <ul> <li>Windows releases are now 64 bit by default (32 bit releases remain available)</li> <li>A fix for the macOS black-screen startup freeze after Mono or macOS upgrades</li> <li>Future support for the .NET Core runtime (this work is still in progress)</li> <li>Improved development tooling and IDE support for OpenRA and the <a href="https://github.com/OpenRA/OpenRAModSDK">Mod SDK</a></li> </ul> <p>Playtest releases do not overwrite the OpenRA release version, and you can automatically switch between the two versions when joining a multiplayer server.
