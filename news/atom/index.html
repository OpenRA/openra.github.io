RA] Saboteur / Tanya can now place C4 on vehicles.</li> <li>[RA &amp; TD] Tanya / Commando can now place C4 on bridges.</li> <li>[TD] Added several of the original campaign missions.</li> <li>[RA] Added the Allies Phase Transport and and Soviet Hijacker units.</li> <li>[RA &amp; TD] A number of balance changes (see the <a href="https://github.com/OpenRA/OpenRA/blob/playtest-20140405/CHANGELOG">full changelog</a> for details).</li> </ul> <p>Changes that impact mod and custom-map authors include:</p> <ul> <li>Improvements to the Lua scripting API.</li> <li>Removal of the legacy scripted missions.</li> <li>The legacy (RA &amp; TD) map converter has moved from the map editor into OpenRA.Utility.exe.</li> <li>Added <span style="font-family: monospace;">--upgrade-mod</span> and <span style="font-family: monospace;">--upgrade-map</span> utility commands to automatically upgrade a mod or map targeting release-20131223 to the latest version.</li> <li>A number of trait properties have changed from pixel or cell units into world units. Use –upgrade-mod to automatically convert these.</li> </ul> <p>A significant issue was discovered with our Lua integration late in this playtest cycle. This shouldn’t significantly affect scripted missions, but caused the shellmaps (the animations that run in the main menu) to become unstable after some time. Rather than delay the playtest further, we have disabled the shellmaps until the issue has been fixed. The shellmaps will be restored before the next full release.</p> <hr /> <p>We have also been nominated for the Linux Game Awards Project of the Month for May 2014.</p> <div style="text-align:center"> our <a href="http://www.linuxgameawards.org/award-instance/project-of-the-month-may-2014">competitors</a>, they are great projects too!).</p> <hr /> <p>A brief taste of our new playtest changes can be
