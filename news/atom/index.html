http://www.openra.net/ OpenRA - News Feed 2017-07-21T20:00:00Z OpenRA developers http://www.openra.net tag:www.openra.net,2017-07-21:/news/playtest-20170722/ Playtest 20170722 out now! 2017-07-21T20:00:00Z 2017-07-21T20:00:00Z <p>We’re pleased to finally announce playtest-20170722, the first release candidate for what will be the biggest OpenRA release in many years. Core development has focused on three areas, and we have included many other great improvements contributed by our open source community.</p> <p>You’ll notice the first major change before you even start the playtest: the “OpenRA” launcher is no more, and has been replaced with individual launchers for Red Alert, Tiberian Dawn, and Dune 2000. This is part of a major reorganisation of the project structure to improve backwards compatibility and modding support.</p> <div style="text-align:center"> <p><img src="/images/news/20170722-launchers.png" alt="New launchers" /></p> <p>Individual mods are now launched directly from your Desktop / Application Launcher.</p> </div> <p>The in-game mod chooser has been removed, but it is still possible to switch mods when joining a server from the Multiplayer list. Even better, if you have multiple versions installed (e.g. Playtest and Release, which now install to different locations on Windows) the server list will show games from <em>all</em> versions, and will switch to the correct version when you try to connect.</p> <p>The second focus area relates to how you control your units on the battlefield. For many years we have supported features like Attack Move, unit stances and queuing orders using hotkeys, but these have not always been obvious or intuitive to use. We have added a long-planned command bar interface for these, and made some adjustments and bug fixes to make them better and behave more like the original games.</p> <div style="text-align:center"> <p><img src="/images/news/20170722-ra-commandbar.png" alt="Unit command bar" /></p> <p>A new bottom command bar is now available for issuing common orders and changing unit stances.</p> </div> <p>The final focus area fundamentally changes the way that buildings occupy space and receive damage, taking inspiration from the original Dune 2000, which was more advanced than the earlier (and some later!) games.</p> <div style="text-align:center"> <p><img src="/images/news/20170722-td-buildinghitshapes.png" alt="Building hit boxes" /></p> <p>New building physics improves the way that structures are targeted and take damage.</p> </div> <p>Some of the smaller but still important new features include:</p> <ul> <li>Automatic discovery of LAN games in the Multiplayer browser.</li> <li>Improvements to the Global Chat UI.</li> <li>New search and filtering options in the Map editor.</li> <li>Another 9 campaign missions for D2K, which is now half way complete!</li> <li>Missing maps can now be installed directly from the Replay browser.</li> <li>Significant performance and memory improvements when playing music.</li> </ul> <p>Please see the <a href="https://github.com/OpenRA/OpenRA/wiki/Changelog/f4a27f19459d2dd35d4bc08a75a741363f146422">full changelog</a> for information on many more improvements that we have not mentioned here.</p> <div style="text-align:center"> <p><img src="/images/news/20170722-d2k-editor.png" alt="New D2K missions" /></p> <p>It is now possible to search and filter actors and tiles in the map editor.</p> </div> <p>This playtest includes big changes which may have broken your favourite feature, and which have definitely affected the game balance in all three mods. This makes it more important than ever to play games on this and future playtests and let us know of any issues via <a href="http://www.sleipnirstuff.com/forum/viewforum.php?f=80">our forum</a>, our <a href="https://github.com/OpenRA/OpenRA/issues">GitHub issue tracker</a>, or in the comments below.</p> <p>Don’t forget, if you’re on Windows or macOS you can have playtest-20170722 and release-20170527 installed at the same time, and switch between the two when joining servers with other players. So head over to our <a href="/download/">download page</a> and give it a try!</p> <div class="about-todo-divider"></div> <p>And now for a special note to our modding community:</p> <p>As OpenRA modding grew in popularity it became clear that requiring players to manually install mods <em>on top of</em> our OpenRA releases, with the in-game mod chooser to switch between mods, was more of a burden than a benefit. We have listened to the feedback from both players and modders, and have spent the last few months working on a comprehensive solution.</p> <p>We have done this by decoupling the game engine (OpenRA.Game.exe and support files) from our default mods, and have put <em>the mod</em> in charge. Mods now manage their own engine files, which means that they can exist <em>alongside</em> our official mods with no dependencies between them.</p> <p>Our new <a href="https://github.com/OpenRA/OpenRAModSDK">mod SDK</a> provides a development environment that simplifies the engine and dependency management, and allows you to generate proper installers for players to install your game. Games built with the mod SDK will not be affected when the player upgrades or even uninstalls other OpenRA-based games.</p> <p>The mod SDK is still a new project, and our documentation is still rough. We encourage modders who want to port their projects over to the SDK to file issues if you have any problems, or visit our <a href="irc://chat.freenode.net/openra">development IRC channel</a> (<a href="http://webchat.freenode.net/?channels=openra">webchat</a>) if you have any questions during european-timezone evenings.</p> <div class="about-todo-divider"></div> tag:www.openra.net,2017-06-19:/news/10th-anniversary/ 10 years of OpenRA! 2017-06-19T19:00:00Z 2017-06-19T19:00:00Z <p>We are proud to celebrate the 10th anniversary of OpenRA today!</p> <p>Community member SoScared put together a new fantastic promotional trailer:</p> <div style="text-align:center"> <iframe width="560" height="315" src="https://www.youtube.com/embed/93b7TKooj-M" frameborder="0" allowfullscreen=""></iframe> </div> <hr /> <p>Meanwhile, we are working towards a new playtest.
