Posted by Paul Chote on 11 February 2014

DNS Issues

Our website DNS provider is experiencing sporadic problems, which is affecting new content downloads, our online server listings, and automatic map downloads.

We are working on resolving this issue, but this may take some time.

Manually installing content

If you receive a "Can't connect to remote server" error in game, then you can manually install the content:

  • Download the content assets for your favorite mod: Red Alert, Tiberian Dawn, Dune 2000.
  • Extract the zip file contents into the following directory (where <MOD> is either ra, cnc, or d2k)
    • Windows Users: My Documents\OpenRA\Content\<MOD>
    • Mac OSX Users: ~/Library/Application Support/OpenRA/Content/<MOD>
      (The Library folder is hidden, so use the Finder's Go → Go To Folder... option and paste this path.)
    • Linux Users: .openra/Content/<MOD>

Manually finding games

If the server list is empty or gives an error, you can still direct connect to games manually.

Players are using our IRC channel to organize games via direct connect.

Manually installing maps

The automatic map downloads are not functioning, so the game will produce an exception and close if you attempt to connect to a server running a custom map that you do not have installed.
You can manually install missing maps to the OpenRA/maps/<MOD> directory indicated above.
